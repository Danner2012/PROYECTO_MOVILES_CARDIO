================================================================================
          DOCUMENTACIÓN TÉCNICA: FLUJO DE INTELIGENCIA ARTIFICIAL
                       PROYECTO CARDIOLOGÍA (RAG)
================================================================================

Este documento explica cómo funciona la integración de la IA (Ollama/Mistral)
con la base de datos de cardiología en el sistema.

--------------------------------------------------------------------------------
1. ARQUITECTURA GENERAL (FLUJO DE DATOS)
--------------------------------------------------------------------------------

El sistema utiliza una técnica llamada RAG (Retrieval-Augmented Generation), 
lo que significa que la IA no responde "de memoria", sino que primero busca 
datos reales en tu base de datos y luego responde basada en ellos.

FLUJO:
App Flutter -> FastAPI (8001) -> PostgreSQL -> Ollama (11434) -> FastAPI -> Flutter

--------------------------------------------------------------------------------
2. COMPONENTES DEL BACKEND (backend/ollama_api/)
--------------------------------------------------------------------------------

* main.py:
  - Es el "cerebro" del backend.
  - Recibe la pregunta de Flutter.
  - Llama a `crud.py` para buscar pacientes en la base de datos.
  - Construye un "Prompt" (instrucción) que incluye los datos médicos encontrados.
  - Envía todo a Ollama y devuelve la respuesta final.

* database.py:
  - Configura la conexión con PostgreSQL usando las credenciales del archivo .env.

* models.py:
  - Define la estructura de la tabla 'cardio_records' en la base de datos 
    (Paciente, Arritmia, Ritmo Cardiaco, Diagnóstico, etc.).

* schemas.py:
  - Define las reglas de validación de datos usando Pydantic. 
    Asegura que los datos que entran y salen de la API tengan el formato correcto.

* crud.py:
  - Contiene la lógica de búsqueda inteligente. 
  - Limpia la pregunta del usuario y busca palabras clave en los nombres de 
    pacientes, diagnósticos o síntomas dentro de PostgreSQL.

* seed.py:
  - Script para "sembrar" la base de datos. Crea 10 registros de ejemplo de 
    pacientes con diferentes condiciones cardiológicas para pruebas.

--------------------------------------------------------------------------------
3. COMPONENTES DEL FRONTEND (flutter_frontend/lib/features/ollama/)
--------------------------------------------------------------------------------

* data/ollama_client.dart:
  - Es el cartero. Se encarga de enviar la pregunta del usuario por HTTP 
    a la dirección http://127.0.0.1:8001/chat (FastAPI).

* logic/ollama_provider.dart:
  - Gestiona el estado del chat. Guarda los mensajes que se ven en pantalla,
    maneja el indicador de "Ollama está respondiendo..." y actualiza la vista.

* presentation/screens/ollama/ollama_screen.dart:
  - Es la interfaz de usuario (el chat).
  - Diseñada para administradores.
  - Incluye burbujas de chat, campo de texto y scroll automático.

--------------------------------------------------------------------------------
4. CONFIGURACIÓN EXTERNA
--------------------------------------------------------------------------------

* Ollama (Mistral):
  - Motor de IA local. Debe ejecutarse con OLLAMA_ORIGINS="*" para permitir
    que el navegador Chrome hable con él.

* PostgreSQL:
  - Base de datos donde residen los 10 registros médicos que la IA consulta.

--------------------------------------------------------------------------------
5. RESUMEN DE UNA CONSULTA
--------------------------------------------------------------------------------

Usuario pregunta: "¿Quién es Juan Pérez?"
1. Flutter envía "¿Quién es Juan Pérez?" a FastAPI.
2. FastAPI busca "Juan" y "Pérez" en la tabla 'cardio_records'.
3. FastAPI encuentra: "Juan Pérez tiene Taquicardia y ritmo de 110 BPM".
4. FastAPI le dice a Mistral: "Usa este dato: [Juan Pérez, 110 BPM, Taquicardia] 
   y responde la pregunta: ¿Quién es Juan Pérez?".
5. Mistral responde: "Juan Pérez es un paciente registrado con Taquicardia...".
6. Flutter muestra la respuesta en el chat.
